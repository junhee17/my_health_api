# 個人食事・運動記録API (ポートフォリオ)

## 1. プロジェクト紹介

このプロジェクトは、バックエンド開発の全体的な流れを学習することを目標に進行しました。

学習のテーマとして、既存の食事・運動記録アプリから様々な付加機能を取り除き、「記録」という本質的な機能だけに集中したミニマルなサービスはどのような形になるだろうか、という発想から出発しました。

このシンプルなコンセプトを通じて、CRUDやデータベース設計といったバックエンドの核心技術の学習に完全に集中することができ、一つのアイデアを技術で具現化していく過程を経験しました。

## 2. 主な機能

- 日付別の食事・運動記録の**作成(Create)、照会(Read)、更新(Update)、削除(Delete)**が可能なFull CRUD APIを実装しました。

## 3. 使用技術

- **Language:** Python
- **Framework:** FastAPI
- **Database:** SQLite, SQLAlchemy (ORM)
- **API Test:** FastAPI Docs (Swagger UI), Thunder Client

## 4. 開発プロセスと学習方法

開発の全過程において、AIを積極的に活用しました。

私はこのプロジェクトの「設計者」であり、「質問する人」でした。まず、プロジェクトの目標と構造を私が直接構想した後、具体的なコードの実装や初めて接する概念についてはAIに質問しながら答えを得る方式で進行しました。

単に動作するコードを得るだけでなく、「このコードはなぜこのように動作するのか？」「このエラーはなぜ発生するのか？」のように、根本的な原理を理解するためにAIを自分だけの「家庭教師」のように活用しました。

## 5. データベース関係モデル

- 「一日の記録(DailyLog)」と「食事(Food)」/「運動(Workout)」の関係を**1:N(一対多)**関係として設計しました。
- 一つの日付には、複数の食事及び運動の記録を持たせることができます。

## 6. ローカルでの実行及びテスト方法

### 6-1. プロジェクトフォルダへの移動
> このプロジェクトコードをダウンロードした後、ターミナルを開き、以下のコマンドでフォルダに移動します。
```bash
cd path/to/my-health-api
```

### 6-2. 仮想環境の作成と有効化
> 以下の二つのコマンドをターミナルで順番に実行します。
```bash
python3 -m venv venv
source venv/bin/activate
```

### 6-3. 必要なライブラリのインストール
> requirements.txt ファイルを利用して、全てのライブラリを一度にインストールします。
```bash
pip install -r requirements.txt
```

### 6-4. サーバーの実行
```bash
uvicorn main:app --reload
```

### 6-5. APIのテスト
- サーバーが実行された後、以下のURLをウェブブラウザにコピーしてアクセスすると、全てのAPIを簡単にテストできる対話型のドキュメントページが開きます。
[http://127.0.0.1:8000/docs](http://127.0.0.1:8000/docs)
